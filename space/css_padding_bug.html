This is an old test case for firefox bug 270241, which was fixed in Gecko build 20050428.


This div alternates between including and not including right-side padding, even though the CSS only specifies left-side padding. This happens during interactive use of the browser, such as changing text size and find-as-you-type.

Try it! Press Control-Plus; the right-side padding should disappear. Now type 'test to highlight the link. The right-side padding should reappear.
